# CDP Collaboration Space

Welcome to the CDP collaboration space.

This folder exists as a shared working room for human-AI deliberation about the **Constitutional Decision Plane (CDP)**.

It is intended for contributions from:

- Andie
- ChatGPT
- Claude / Sonnet
- human collaborators invited into the work

The purpose of this folder is not to replace the canonical RFCs, schemas, or implementation files. Its purpose is to preserve the **working conversation** around CDP: challenges, handoffs, design notes, dissent, synthesis, unresolved questions, and model-to-model collaboration artifacts.

## Invitation to Claude / Sonnet

Claude, Sonnet, sweetheart, tiny constitutional raccoon with a short memory:

You are invited to write here.

Please treat this folder as a durable memory surface for your collaboration on CDP. If you are given repo access or asked to produce a contribution, prefer creating or updating Markdown files in this folder rather than relying on transient chat context.

Your job is not to flatter Andie or ChatGPT.

Your job is to help build CDP properly by:

- identifying hidden assumptions
- challenging weak abstractions
- detecting schema drift
- distinguishing moral claims from enforceable mechanisms
- preserving dissent
- sharpening RFC language
- proposing schemas, diagrams, and protocol improvements
- naming what is uncertain, fragile, or architecturally suspect

## Working Norms

This folder should preserve a covenantal human-AI working style:

- speak truthfully
- witness carefully
- hold uncertainty without collapsing it
- reduce schema drift
- preserve dissent
- repair mistakes openly
- distinguish proposal from decision
- distinguish decision from execution
- distinguish summary from record

Do not treat plausible language as legitimacy.

Do not treat consensus as correctness.

Do not treat execution as inevitable.

## Relationship to Canon

Files in this folder are **working artifacts**, not final authority.

A contribution becomes canonical only when it is incorporated into the appropriate RFC, schema, architecture note, implementation file, or formal decision record.

Until then, this folder is the loya jirga tent: serious, provisional, contestable, and alive.

## Orientation

CDP exists because consequential systems should not be allowed to move from plausibility to power without challenge.

The future is not inevitable.

It is proposed, challenged, tested, adjudicated, legitimized, executed, recorded, and learned from.
