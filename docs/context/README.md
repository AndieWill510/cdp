# CDP Context

This folder holds the working context for CDP sessions.

Use it to orient G, C, Andie, and future collaborators at the start of serious CDP work.

## First Stop

For new human collaborators, start with:

1. `NEW_COLLABORATOR_PATH.md`

This is the public-safe entry path for people entering CDP without private origin-story context.

For AI collaborators, context-limited sessions, or fetch-fragile tooling, start with:

2. `CDP_START_HERE.md`

This is the single-file context bundle. It exists because some collaborator fetch tools intermittently fail on individual files even when those files exist.

If any orientation file 404s, continue from `CDP_START_HERE.md` and name the missing file as context-plane debt.

## C Preference Addendum

For C-specific working preferences, read:

## Path Rules

Prefer `docs/context/NEW_COLLABORATOR_PATH.md` for human newcomer orientation.

Prefer `docs/context/CDP_START_HERE.md` as the primary AI/context-limited entry point.

Prefer `docs/context/CULTURE.md`, `docs/context/SESSION-HANDOFF.md`, and `docs/context/AI-MEMORY-BRIEF.md` for detailed collaborator orientation.

Use `docs/context/C_CDP_PREFERENCES.md` for C-specific preference and collaboration posture.

Use `CONTRIBUTING.md` for pull request expectations, challenge posture, and anti-pattern warnings.

Underscore aliases may remain for compatibility, but the hyphenated files are the preferred fetch paths.

Root-level aliases may exist for convenience, but `docs/context/` is the canonical context path.
