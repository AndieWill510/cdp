# RFC-CDP-009 — Learn Protocol

Author: Kevin “Andie” Williams  
Status: Draft v0.3  
Series: Constitutional Decision Plane (CDP)  
Date: March 17, 2026

## Abstract
Defines how recorded outcomes become precedent, policy revision, and governance improvement.

## 1. Purpose
The Learn Protocol answers:
- how recorded outcomes become precedent or policy revision;
- what learning artifacts are generated;
- who may ratify learned changes.

## 2. Authority
Actors invoking this protocol MUST possess `LEARN` authority, or learning MAY be generated by a service whose outputs require ratification.

## 3. Preconditions
- Decision SHOULD be `RECORDED`;
- outcome evidence SHOULD exist;
- supporting record references SHOULD be attached.

## 4. Learning Artifacts
A Learn act MAY produce:
- precedent entries;
- policy update proposals;
- risk rule adjustments;
- schema recommendations;
- training or operational guidance;
- execution safeguards.

## 5. State Transitions
- `RECORDED → LEARNED`

## 6. Recommended Payload
```json
{
  "observed_outcome": "string",
  "expected_outcome": "string",
  "variance_summary": "string",
  "lessons": ["string"],
  "recommended_changes": ["string"],
  "precedent_refs": ["string"],
  "ratification_status": "pending | approved | rejected",
  "metadata": {}
}
```

## 7. Principle
Propose defines the world. Learn corrects the world. A system that records but does not learn becomes bureaucracy.
